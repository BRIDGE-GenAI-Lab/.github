# Contributing to BRIDGE GenAI Lab

Thank you for your interest in contributing! Whether you're fixing a bug, improving documentation, or proposing a new experiment, we appreciate your help in making AI agents safer.

## Getting Started

1. **Fork** the repository you want to contribute to
2. **Clone** your fork locally
3. **Create a branch** for your work: `git checkout -b feature/your-feature-name`
4. **Make your changes** and test them
5. **Push** to your fork and **open a Pull Request**

## Types of Contributions We Welcome

### Code Contributions
- Bug fixes and performance improvements
- New attack scenarios or defense mechanisms
- Evaluation harness extensions
- Test coverage improvements

### Research Contributions
- Reproducing and validating our published results
- Extending experiments to new models or configurations
- Proposing new evaluation metrics or benchmarks

### Documentation
- Improving README files, tutorials, or API docs
- Adding usage examples
- Translating documentation

### Bug Reports
- Open an issue with a clear title and description
- Include steps to reproduce, expected vs. actual behavior
- Specify your environment (OS, Python version, package versions)

## Code Standards

- **Python**: Follow PEP 8. We use `black` for formatting and `ruff` for linting.
- **Commits**: Write clear, concise commit messages. Use the imperative mood ("Add feature" not "Added feature").
- **Tests**: Include tests for new functionality. Ensure existing tests pass before submitting.
- **Documentation**: Update relevant docs when changing functionality.

## Pull Request Process

1. Ensure your PR targets the `main` branch (unless otherwise specified)
2. Write a clear PR description explaining **what** and **why**
3. Link any related issues
4. Ensure CI checks pass
5. Request a review from a maintainer

We aim to review PRs within **one week**. For larger contributions, consider opening an issue first to discuss the approach.

## Citing Our Work

If you use our code or build upon our research, please cite the relevant papers. BibTeX entries are available in each repository's README.

## Security Vulnerabilities

If you discover a security vulnerability, **do not open a public issue**. Instead, follow our [Security Policy](SECURITY.md) and report it privately.
